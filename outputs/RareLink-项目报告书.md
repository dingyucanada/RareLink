# RareLink（稀联）项目报告书

> **项目定位：** 面向罕见病及小样本多中心医学影像科研的可信 Agent 与联邦研究平台。
> **项目状态：** 研究用途工程原型；不提供诊断或治疗建议，不宣称临床有效性或真实多医院生产部署。
> **开源地址：** [github.com/dingyucanada/RareLink](https://github.com/dingyucanada/RareLink)

---

## 一、项目概述

### 1. 项目名称

**RareLink（稀联）——数据不出院、证据可回溯的多中心科研智能体平台。**

### 2. 项目目标

RareLink 的目标不是再提供一个孤立的医学影像训练脚本，而是帮助医院科室和多中心研究团队在原始数据不能集中时，仍然形成可协作、可审计、可复核、可持续迭代的科研证据。

系统把研究协议、站点可行性、实验合同、本地训练、联邦聚合、隐私审阅与研究写作组织为受控闭环：**数据留在科室，模型在本地训练，跨站只交换获批准的更新与聚合指标，关键过程与边界写入审计账本。**

### 3. 项目背景

罕见病、儿童肿瘤和其他小样本医学影像研究普遍面临四类问题：

1. **数据孤岛。** 原始 MRI、标注、患者字段与 DICOM 标识不能被随意汇集；
2. **站点异质性。** 多中心病例量、扫描协议和病灶负荷存在差异，只看平均指标会掩盖弱站点风险；
3. **研究治理不足。** 方案、数据划分、主要终点、训练预算与报告边界若依赖口头约定，难以复现和审计；
4. **Agent 使用边界不清。** 大语言模型可以辅助协议与写作，但不能接触原始影像、患者信息或越权输出诊疗建议。

RareLink 以儿童高级别胶质瘤多中心 MRI 分割作为长期应用场景，并以公开 MSD `Task01_BrainTumour` 数据完成外部工程兼容性验证。当前赛事版本在一台 NVIDIA DGX Spark 上模拟三个**逻辑站点**，用于验证计算、联邦、策略和证据链，不把它包装成三家真实医院或临床研究结论。

## 二、作品介绍：功能与亮点

### 1. 从“训练任务”升级为“研究证据闭环”

RareLink 将一次多中心研究组织为以下工作流：

```text
研究问题 → 结构化协议 → 站点可行性 → 实验合同与人工锁定
→ 本地训练 / 联邦聚合 → 指标与模型收据 → 统计和隐私复核 → 研究报告与证据包
```

每一步均有状态、输入、输出和审计记录。失败任务可重试，但不会静默覆盖既有证据或生成重复实验。

### 2. 证据驾驶舱

React 前端将“可交互的研究流程沙盘”与“已落盘的实机收据”明确分开：

- 流程沙盘呈现协议、可行性、合同、Agent 和审批状态；
- 证据驾驶舱读取公开影像工程运行收据，展示 `3/3` 聚合、全局模型、站点 Dice、HD95、训练耗时和非临床边界；
- 用户可点击核验本地证据哈希，再展开站点明细；
- 页面把“服务在线”“回执已采集”“独立核验通过”分层呈现，无真实证据时显示 `NOT CLAIMED`。

### 3. 联邦研究与站点公平性

系统支持 Local、FedAvg、FedProx、严格 SVT 与样本级 DP-SGD 等策略对照。除了平均 Dice，还固定呈现最弱站点 Dice、站点离散度和 HD95，避免“平均分提升、个别站点退化”被忽略。

单 Spark 原型使用统一内存保护锁串行运行三个三维逻辑站点，避免多模型竞争资源；真实落地时，每家医院部署独立 Spark Client，协调方仅聚合经合同批准的更新和指标。

### 4. 五角色 Agent Team

| Agent 角色 | 主要职责 | 安全限制 |
| --- | --- | --- |
| 研究主任 | 将自然语言研究问题转为结构化协议 | 不读取影像、标签或病例字段 |
| 实验设计 | 提出 Local/FedAvg/FedProx 等比较合同 | 不能自行锁定合同或改变主要终点 |
| 统计评审 | 解释平均/最弱站点指标与局限性 | 不将单次工程结果写成医学结论 |
| 隐私评审 | 检查报告草稿和发布策略 | 可阻断报告，不可放宽外发政策 |
| 科研写作 | 基于已批准证据生成研究叙事 | 不生成诊疗建议或虚构实验结果 |

### 5. 已完成的工程验证

| 验证项 | 实际结果 | 声明边界 |
| --- | --- | --- |
| 公开影像工程验证 | MSD Task01 24 例四模态 MRI 完成归档/几何/哈希校验、单站 CUDA 训练、三逻辑站点一轮 FedAvg；`3/3` 更新聚合并持久化全局模型 | 工程兼容性验证；非儿童队列、非临床性能、非真实跨院验证 |
| 稳定性对照 | 5 种子 × 5 策略 × 3 轮，`25/25` 组合完成 | 合成数据与逻辑站点的工程比较，不作医学统计推断 |
| DP-SGD | Opacus 样本级 DP-SGD，三轮保守会计 `ε=6.076881`、`δ=1e-5` | 仅覆盖本地训练步骤，不是端到端、用户级或医院级 DP |
| 安全通信 | Spark–Mac mTLS 注册、重连、错误身份拒绝 | 安全联邦演练，不等同医院生产网络或身份体系 |
| Agent 安全 | 12 条输入与 14 条输出固定门控用例，`26/26` 通过 | 验证策略网关，不是完整渗透测试或通用模型安全认证 |

## 三、技术创新点说明

### 1. 联邦学习的“证据闭环”产品化

传统联邦学习项目通常只交付模型或指标。RareLink 将协议、可行性、合同、任务状态、模型路径、聚合指标、失败与审计事件串联为可追溯状态机，使研究团队可以回答“结果从何而来、谁批准过、能否复跑、在哪些边界下成立”。

### 2. 影像计算边界与语言模型边界分离

原始 MRI 与标签留在站点；DGX Spark 执行 MONAI/PyTorch 本地三维训练；NVIDIA FLARE 只协同获批准更新；Step 3.7 或 Spark 本地 TensorRT-LLM 仅接收策略过滤后的脱敏文本和聚合指标。该分层避免把“Agent 能力”误当成对患者数据的访问权限。

### 3. 将最弱站点纳入主要研究证据

系统同时记录平均指标、最弱站点指标、站点方差与 HD95。对于多中心研究，这种设计比单一平均分更能发现站点间风险，也为后续公平性审查、试点准入和科研报告提供了可操作依据。

### 4. 隐私—效用可比较，而非口号化

RareLink 将 Local、FedAvg、FedProx、严格 SVT、DP-SGD 放入同一实验合同和多种子矩阵，记录成功、退化与资源代价。样本级 DP-SGD 使用逐样本裁剪、高斯噪声、Poisson 采样和 RDP 会计；隐私预算及其适用边界均在报告中明确说明。

### 5. Agent 的双向安全门控与确定性回退

输入门阻断原始影像、标识符、DICOM UID、路径、密钥和小样本字段；输出门阻断诊疗指令、患者数据请求、临床夸大与数据合同提权。远程模型不可用时，系统退回确定性模板 Agent，保证科研控制面可运行、可测试，而不是因外部 API 失效中断。

### 6. 本地大模型证据不“补造”

TensorRT-LLM 本地路径区分端点在线、运行回执采集和独立验证三个状态。只有同时存在真实 GPU 快照、无内容元数据回执和本地网关红队结果，页面才会显示已核验；否则明确为 `NOT CLAIMED`。这是对本地大模型能力声明的可审计约束。

| 外部工程验证 | 在合规授权的 BraTS-PEDs 等儿童公开数据上重复工程流程 | 遵守数据使用政策、完成预处理、引用和透明报告 |
