# RareLink 三分钟真人口播分段录音稿

> 请逐段录制，每段开始和结束各留约两秒安静环境声。说错时停一下，重新说整句即可，不需要追赶固定时间。

## 第一段｜开场

大家好，我们是“发现新的大陆”团队。这是 RareLink 稀联：面向罕见病多中心科研的隐私保护智能体系统。我们的原则只有一句话：数据不动，知识流动。

## 第二段｜研究问题

罕见病研究的核心矛盾，不是没有算法，而是病例分散、样本稀少、数据难以跨院流动。全球已知罕见病超过七千种，单病种样本往往很少。儿童高级别胶质瘤，正是一个需要多中心协作、又必须保护影像隐私的典型场景。

## 第三段｜系统架构

RareLink 采用四层架构。底层数据留在科室；DGX Spark 承担三维医学影像计算；NVIDIA FLARE 只交换模型更新；上层智能体团队负责研究协议、实验编排、证据审计和报告。Step 模型只读取脱敏协议与聚合指标，不接触原始影像。

## 第四段｜智能体团队

五个智能体不是自由聊天，而是有明确权限边界的科研团队。数据管家检查可用性，分析智能体设计实验，证据智能体校验证据，报告智能体生成可追溯叙事，编排智能体管理状态。涉及研究合同和发布结论时，必须由人类研究者审批。

## 第五段｜证据闭环

每一次实验都进入证据闭环：研究问题先被转换成版本化协议，再生成不可静默修改的实验合同。训练完成后，系统记录指标、日志、模型哈希和审计事件。现在我们点击“核验本地证据哈希”，系统重新读取已提交的聚合文件，返回完整性结果和三站点收据。前端展示的数字，不是只存在于演示页面。

## 第六段｜DGX Spark 实机能力

DGX Spark 在这里不是网页服务器。我们在真实 GB10、ARM64 与 CUDA 十三环境，运行 CUDA 计算、MONAI 三维 SegResNet、NVIDIA FLARE 二点七点二，以及 RareLink 的 API 和证据驾驶舱。二十四例 MSD 公开四模态脑肿瘤 MRI 已在 Spark 完成校验和本地训练，三个逻辑站点的更新全部聚合，端到端约六十九秒。网页把这一真实收据和可点击的工作流沙盘明确分开。

## 第七段｜关键实验与安全证据

为了避免只展示一次漂亮结果，我们完成了五个随机种子、五种策略、三轮联邦的重复实验；加入样本级差分隐私训练与隐私会计；完成 Spark 和 Mac 两台物理设备的 mTLS 正向与负向演练；并用二十六条固定用例红队测试智能体安全网关。真实 MSD 运行则是每站一轮、一 epoch 的工程验证，不把它包装成临床结论。所有关键结果都进入证据清单。

## 第八段｜隐私与公平性

隐私不是一句口号。RareLink 同时使用本地留存、最小化汇总、差分隐私、安全聚合设计、访问控制和全链路审计。我们报告平均 Dice，也报告最弱站点、站点差异和 HD 九十五，防止平均值掩盖小中心的性能退化。当前结果只证明工程链路，不代表临床有效性。

## 第九段｜一键复现

评委可以通过一个命令运行核验包。它不下载医学影像，也不需要 API 密钥，会检查五种子实验、样本级 DP 声明边界、两设备 mTLS 和智能体红队四个必检门。四项全部通过，结果还可以与证据驾驶舱中的卡片逐项对应。

## 第十段｜诚实边界与下一步

我们也主动标出项目边界：公开 MSD 影像工程验证已经完成，但它不是儿童队列、临床有效性验证或真实多医院生产部署；当前仍是一台 Spark 上的三个逻辑站点。本地大模型路径已经实现，但在没有真实 Spark 推理回执以前，绝不把它写成已经完成。下一步是授权数据、真实多节点和临床科研合作。

## 第十一段｜结尾

RareLink 不是另一个脑肿瘤分割演示，而是一套让数据不出院、证据可回溯的多中心科研操作系统。让智能体发现医学证据，也让每一项结论经得起复现和审计。谢谢。
