# DGX Spark 黑客松十日谈：把“联邦学习脚本”做成科研证据系统

## Day 1：从宏大命题收缩到可信问题

最初的想法是“罕见病联邦学习平台”。它听起来完整，却几乎无法在十天内验证。我们最终聚焦儿童高级别胶质瘤多中心 MRI 分割：小样本、四模态三维影像、站点异质性和数据不能集中这四个矛盾，恰好能让 DGX Spark、MONAI、NVIDIA FLARE 与 Step 3.7 各自承担不可替代的职责。

## Day 2：先划边界，再写功能

医疗方向最危险的不是模型效果低，而是把工程演示说成临床结论。我们先写下三条不可突破的边界：比赛版的三个站点是单 Spark 上的逻辑模拟；原始影像、标签与患者级字段不进入 Agent；任何结果都只是一项研究工程证据。随后把“人工审批”“受控字段”“可审计账本”放进状态机，而不是写进 README。

## Day 3：让 Agent 有分工，也有约束

我们没有让一个聊天模型承担所有事，而是拆成研究主任、站点数据管家、实验设计、统计评审、隐私评审和科研写作角色。每个角色输出 Pydantic 校验的结构化对象；实验合同锁定策略、划分种子、预算、主要终点和隐私边界。Step 3.7 只接收协议和聚合指标，网络不稳定时系统退化为确定性模板 Agent，保证科研流程不被云端依赖卡住。

## Day 4：真实跑通三维训练，而非模拟指标

我们生成了四模态 NIfTI 合成队列，以便不接触任何真实患者数据也能验证整个工程管道。MONAI 读取影像、SegResNet 前向与反向传播、DiceCE 损失、Dice/HD95 指标、模型保存全部实际执行。合成数据和单轮训练的分数很低，但这正是应当展示的诚实边界：它验证链路，不证明临床效果。

## Day 5：联邦学习不是一个按钮

NVIDIA FLARE 负责真实客户端更新和聚合。我们实现三个逻辑站点的 Local、FedAvg 与 FedProx，对每站点保留指标并计算平均 Dice、最弱站点 Dice、站点标准差和 HD95。最重要的改变是：不再只问“平均分有没有更高”，而是问“平均提升是否以最弱站点退化为代价”。

## Day 6：在真实 DGX Spark 上处理现实问题

节点是 ARM64 的 NVIDIA DGX Spark GB10。CUDA、MONAI 和 FLARE 均成功运行，但 Docker daemon 的代理不可用，缓存环境还对新算力能力给出兼容性警告。我们没有修改共享节点 daemon，也没有把大模型或数据经 SSH 推上去；而是复用可用的本地容器、在节点内安装依赖，并把实际限制写入验证报告。

## Day 7：把任务执行变成可恢复的系统

训练不再是前端点击后等待一个终端结果。FastAPI 后台任务把排队、运行、进度、日志、全局模型路径和异常写入数据库；Spark 统一内存保护锁让多个三维任务串行执行。服务重启会把中断任务转成明确的可重试失败。我们还加入了安全的故障注入开关，用于演示“失败被记录、不会处理数据、可以重新执行”。

## Day 8：数据集选择比“下载更多”更重要

我们实现了 MSD 脑肿瘤公开基准的节点直连下载、MD5/SHA-256、标签映射和三站点非 IID 分区。然而 Spark 到官方源的实测吞吐只有几十 KiB/s，完整下载需要数天。这个问题提醒我们：比赛中最重要的是可信证据，不是把未完成的数据下载包装成成果。于是保留自动化基准链路，把主演示放回已实测的 Spark 工程闭环。

## Day 9：把技术事实变成评委能看懂的证据

我们重做控制台为“证据驾驶舱”。评委首先看到数据留在科室、Spark 本地训练、FLARE 聚合和 Agent 解读的 90 秒路径；随后看到策略公平比较、训练任务、审计时间线和 Agent 资产。新增证据叙事 Agent 从已完成实验的聚合指标生成候选策略、最弱站点风险和局限性，输入边界也被记录进账本。

## Day 10：不把原型说成产品终点

RareLink 已经是一个在真实 DGX Spark 上运行的科研基础设施原型，但不是临床系统。接下来需要真实多院部署、授权数据、外部站点验证、正式隐私会计、身份与证书体系。十天的价值不在于假装完成这些，而在于做出一条诚实、稳定、可扩展的“协议到证据”路径：数据不出科室，模型跨院学习，结论始终可以回到实验账本。
