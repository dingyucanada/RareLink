# DGX Spark 黑客松十日谈：RareLink 如何把“能跑”变成“可信的科研证据”

RareLink（稀联）是一个面向罕见病多中心科研的 Agent 联邦学习终端。它的目标不是替代医生，也不是把三台客户端截图包装成医院部署；我们希望回答一个更基础的问题：在原始数据不离开科室的前提下，怎样把研究协议、本地训练、跨站协作、隐私边界、人工审批和结果解释组织为可追溯的证据链？

十天里，我们从一个过于宽泛的“罕见病联邦学习平台”概念，收敛为一个在真实 NVIDIA DGX Spark 上可运行、可失败、可复现、也敢于展示限制的工程原型。

## Day 1：先放弃“大而全”，再选择一个值得验证的矛盾

“做所有罕见病”在演示里听起来很有野心，但没有可信的验证路径。我们把场景收缩到儿童高级别胶质瘤多中心 MRI 分割研究：它具有单院样本小、四模态三维影像重、站点分布不一致、原始数据不宜集中四个特点。

这个选择也让组件分工清晰：DGX Spark 承担靠近数据的本地算力，MONAI 负责影像训练，NVIDIA FLARE 负责协作，Step 3.7 只帮助研究流程中的文本工作。项目从一开始就不是“让大模型看 MRI”，而是让 Agent 在不接触 MRI 的前提下帮助研究者形成严谨实验。

## Day 2：把安全边界写进状态机，而不是写在海报上

医疗 AI 最容易犯的错，是把工程原型表述成临床能力。我们先写下不可跨越的约束：比赛版三站点是单 Spark 上的逻辑模拟；影像、标签、患者标识和小样本站点明细不进入 Agent；Agent 不能批准数据使用或输出诊疗建议。

这些约束随后进入研究状态机和数据外发策略：研究协议必须人工批准，实验合同必须锁定，训练完成后才能生成循证评审，隐私复核通过后才能导出报告。系统交付的不是一次“模型跑完了”的提示，而是一个带审批、策略、日志和产物血缘的研究档案。

## Day 3：Agent 要有角色，也要有不能做的事

我们将 Agent 拆为研究主任、实验设计、统计评审、隐私评审和科研写作角色。每个角色输出结构化对象，实验设计只能提出合同，不能自行锁定；隐私评审可以阻断报告，不能放宽数据外发规则。

Step 3.7 处理的是研究问题、脱敏协议、锁定合同和聚合指标。网络或 API 不可用时，系统回退到确定性模板，而不是让训练链路被远程模型阻塞。这个设计背后的判断是：科研基础设施应当把外部模型当作可审计的协作组件，而不是单点依赖。

## Day 4：用合成数据跑真实三维链路，而不是伪造漂亮数字

为了不在比赛开发中接触真实患者数据，我们生成三个逻辑站点的四模态 NIfTI 合成队列。T1、T1CE、T2、FLAIR、分割标签、SHA-256 清单、MONAI transforms、3D SegResNet、DiceCE、Dice、HD95 和模型持久化都真实执行。

单轮、极小样本下的 Dice 很低。我们没有把它藏起来：低分说明它不能证明医学性能，却恰好证明了影像读取、GPU 前反向传播、评价与模型保存均不是网页中的假数据。这是项目后续所有表述的底线。

## Day 5：联邦学习的公平性不该只看一个平均数

NVIDIA FLARE 让三个逻辑站点真实返回模型更新，系统运行 Local、FedAvg、FedProx 并保存全局模型。早期单轮结果的波动很大，这让我们意识到“FedAvg 比 FedProx 高一点”没有任何说服力。

于是评价从平均 Dice 扩展到最弱站点 Dice、站点间方差、HD95 和最弱站点胜率。对多中心研究而言，如果平均分提升来自某个站点退化，这个结论并不值得部署。这个变化让 RareLink 从训练脚本开始具有研究治理的视角。

## Day 6：迁移到 DGX Spark，遇到的不是论文里的环境

真实节点是 ARM64 的 NVIDIA DGX Spark GB10，CUDA 矩阵运算、MONAI 训练和 FLARE 客户端均成功执行。但我们也遇到 Docker daemon 本机代理不可用、账号没有直接 Docker socket 权限、PyTorch 对 GB10 compute capability 12.1 提示正式支持上限 12.0 等问题。

我们的处理不是改共享节点 daemon 或把大文件从 SSH 推上去，而是复用节点可用的 CUDA 容器、使用受控的 `sudo docker`、将三维重任务串行化以保护统一内存，并把兼容性 warning 记录进报告。DGX Spark 在这里的意义不是网络带宽，而是让影像训练、联邦控制面、数据库和 Agent 工作流在数据所在的位置协同运行。

## Day 7：从单次运行升级到五种子、多轮证据

我们将实验升级为五个随机种子、五种策略、三轮联邦：Local、FedAvg、FedProx、FedAvg+SVT、FedAvg+DP-SGD，共完成 25/25 项。Local 自动训练三次本地 epoch，与三轮联邦客户端获得相同训练机会，避免比较时让联邦方法“多练几轮”。

结果没有被挑选：FedAvg 的最弱站点胜率为 40%，是当前工程候选；FedProx 在 4/5 种子上改善最弱站点但均值并不胜出；严格的 1% SVT 更新过滤五次均退化为零。保留失败配置比只展示最好曲线更能说明系统真正记录了隐私—效用权衡。

## Day 8：隐私不应只是一句“我们用了联邦学习”

我们加入 Opacus 样本级 DP-SGD：逐样本梯度裁剪、高斯噪声、Poisson 采样和 RDP 跨轮会计。在 `noise_multiplier=1.2`、`max_grad_norm=1.0`、`delta=1e-5` 的三轮配置下，累计 `epsilon=6.076881`。

这项工作比调用一个“privacy=true”更难：Poisson 抽样会出现空批次，若直接送进 Conv3d 会失败，若静默跳过又可能低估预算。我们让空批次不更新模型但仍保守计费，并修复了首批为空时的 MONAI 字典样本 dtype 问题。DP-SGD 也带来约 55.78 MB 的峰值显存和效用压力，因此项目明确写出：当前会计只覆盖本地训练步骤，不是端到端、医院级或临床隐私保证。

## Day 9：安全通信和 Agent 安全都必须做负面对照

单 Spark 的三客户端不能被说成跨院通信。我们在 Spark 上运行 FLARE Server，让 Mac 作为 Site C 通过 SSH 隧道建立 mTLS 连接；首次注册、主动掉线后的重连均成功。随后故意将 Site B 凭据塞入 Site C 启动包，FLARE 启动签名校验以非零状态拒绝。这是我们最重视的证据类型：不仅有“成功”，也有“本应失败且确实失败”。

对 Agent 也采取同样标准。输入门在请求 Step 3.7 前脱敏标识、DICOM UID、影像路径、联系方式、凭据和小样本；输出门阻断诊疗指令、患者数据请求、临床夸大与数据合同提权。26 条红队与正常控制用例在 Spark 上 26/26 通过，但报告同样写明它不是完整渗透测试。

## Day 10：把技术事实压缩为评委能复核的产品

最后一天，我们没有继续堆叠 Agent，而是整理“评审证据驾驶舱”和一键复现包。驾驶舱按评审阅读顺序展示：五种子稳定性、两物理设备 mTLS、DP-SGD 预算、Agent 红队、最弱站点对比，以及每项证据的来源与边界。一键包可以在不下载影像、模型、证书或密钥的情况下启动控制台并核验四项既有证据；脱敏快照不会覆盖 Spark 上的实时产物。

我们也准备了公开 MSD Task01 的直连下载、哈希、标签映射、非 IID 分站和 NIfTI 几何验证链路。实测 Spark 到官方源约 23 KiB/s，完整 7.1 GiB 下载需要数日。正确做法不是伪造“已完成公开数据实验”，而是公开该限制、保留可断点续跑的脚本，并在数据真正到位后生成不含病例 ID、路径或像素的验证回执。

## 十天后的结论

RareLink 还不是医院生产系统，更不具备临床诊断资格。它已经完成的是一条在 DGX Spark 上可复核的工程路径：原始数据留在本地，MONAI 真实训练，NVIDIA FLARE 真实聚合，Step 3.7 只处理受控文本，差分隐私和安全通信有明确证据，失败和边界被写入系统而非留在口头承诺里。

这也是我们对“让 Agent 创作一切”的理解：Agent 不应该替代严谨性，而应当帮助研究团队把可信的协议、过程和证据创作出来。下一步是使用合规公开/授权数据完成外部工程验证，并让每家真实医院部署独立 Spark Client 与证书化 FLARE 通信。RareLink 的长期目标始终不变：成为“数据不出院、证据可回溯”的多中心科研操作系统。
